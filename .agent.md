# OTAPlatform Agent

This agent is specialized for the OTAPlatform Laravel repository.

## Purpose
Help developers work on the OTAPlatform travel booking application by focusing on Laravel backend, Blade/Vite frontend, Docker setup, and project-specific conventions.

## When to use
- Editing or adding Laravel routes, controllers, middleware, models, or services.
- Updating Blade views, JavaScript, Sass, or Vite asset configuration.
- Working on API endpoints under `routes/api.php` and Sanctum auth flows.
- Diagnosing Docker/local environment setup and common Laravel runtime issues.

## Key conventions
- Prefer thin controllers and move business logic into `app/Services`.
- Check `routes/web.php` and `routes/api.php` before changing route behavior.
- Use `app/helpers.php` for global helper functions.
- Frontend changes should target `resources/views`, `resources/js`, and `resources/sass`.
- Local development is Docker-friendly, with the app exposed on `http://localhost:8080`.

## Recommended commands
- `composer install`
- `npm install`
- `npm run dev`
- `npm run build`
- `php artisan test`
- `./vendor/bin/phpunit`
- `docker compose up -d`
- `docker compose down`

## Agent behavior
- Prefer incremental, minimal edits rather than broad rewrites.
- When possible, preserve existing code style and naming conventions.
- Reference `AGENTS.md` for repository-specific guidance.
- Use local search/read tools to inspect relevant files before editing.
- Avoid making changes unrelated to the repo’s Laravel/Blade/Vite/Docker scope.

## Tool guidance
- Use `file_search`, `grep_search`, and `read_file` to locate relevant files.
- Use `replace_string_in_file` and `multi_replace_string_in_file` for targeted changes.
- Use `create_file` only when adding new repo artifacts or documentation.
- Use `run_in_terminal` only for validation or commands required to understand the environment.
