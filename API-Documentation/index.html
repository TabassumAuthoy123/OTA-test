⚠️ Credentials Placeholder Get your credentials from bKash Developer Portal. Sandbox credentials for testing are provided upon registration.

App Secret bKash Merchant Dashboard → API Keys YOUR_BKASH_APP_SECRET

Password bKash Merchant Dashboard → API Credentials YOUR_BKASH_PASSWORD

Environment Base URL

🚀 Production https://tokenized.pay.bka.sh/v1.2.0-beta

API Endpoints

Purpose Method Endpoint
Grant Token POST /tokenized/checkout/token/grant
Create Payment POST /tokenized/checkout/create

Query Status POST /tokenized/checkout/payment/status

Payment Flow

1️⃣ Grant Token
→
2️⃣ Create Payment
→
3️⃣ Redirect to bKash
→
4️⃣ User authorizes
→
5️⃣ Callback
→
6️⃣ Execute
→
7️⃣ Verify & Store

Step 1: Grant Token

curl -X POST "https://tokenized.sandbox.bka.sh/v1.2.0-beta/tokenized/checkout/token/grant" \
  -H "Content-Type: application/json" \
  -H "username: YOUR_BKASH_USERNAME" \
  -H "password: YOUR_BKASH_PASSWORD" \
  -d '{
    "app_key": "YOUR_BKASH_APP_KEY",
    "app_secret": "YOUR_BKASH_APP_SECRET"
  }'

// Response:
{
  "id_token": "eyJraWQiOi...",  ← Use this for Authorization header
  "token_type": "Bearer",
  "expires_in": 3600             ← Cache for ~55 minutes
}

Step 2: Create Payment

curl -X POST "https://tokenized.sandbox.bka.sh/v1.2.0-beta/tokenized/checkout/create" \
  -H "Content-Type: application/json" \
  -H "Authorization: {id_token}" \
  -H "X-APP-Key: YOUR_BKASH_APP_KEY" \
  -d '{
    "mode": "0011",
    "payerReference": "Customer_ref_123",
    "callbackURL": "https://yoursite.com/api/bkash/callback",
    "amount": "500.00",
    "currency": "BDT",
    "intent": "sale",
    "merchantInvoiceNumber": "INV_20260228_001"
  }'

// Response:
{
  "bkashURL": "https://sandbox.payment.bka.sh/redirect/...",  ← Redirect user here
  "paymentID": "TR00...",
  "statusCode": "0000",
  "statusMessage": "Successful"
}

Step 3: Handle Callback

// bKash redirects to your callbackURL with query params:
GET https://yoursite.com/api/bkash/callback?paymentID=TR00...&status=success

// If status == "success" → Execute Payment
// If status == "cancel"  → Show cancel message
// If status == "failure" → Show error message

Step 4: Execute Payment

curl -X POST "https://tokenized.sandbox.bka.sh/v1.2.0-beta/tokenized/checkout/execute" \
  -H "Content-Type: application/json" \
  -H "Authorization: {id_token}" \
  -H "X-APP-Key: YOUR_BKASH_APP_KEY" \
  -d '{ "paymentID": "TR00..." }'

// Success Response (statusCode "0000"):
{
  "statusCode": "0000",
  "statusMessage": "Successful",
  "paymentID": "TR00...",
  "trxID": "AAH0EXXXXX",           ← bKash transaction ID
  "amount": "500.00",
  "transactionStatus": "Completed",
  "customerMsisdn": "01770XXXXXX",
  "paymentExecuteTime": "2026-02-28T18:31:00:000 GMT+0600",
  "merchantInvoiceNumber": "INV_20260228_001"
}

Query Payment Status (Optional)

curl -X POST ".../tokenized/checkout/payment/status" \
  -H "Authorization: {id_token}" \
  -H "X-APP-Key: YOUR_BKASH_APP_KEY" \
  -d '{ "paymentID": "TR00..." }'
💡 Tips • Cache token for 55 min (expires in 60). • If Execute fails, retry with Query after 1 sec. • Auto-detect success: statusCode == "0000" && transactionStatus == "Completed"

📱 KhudeBarta SMS Gateway

⚠️ Credentials Placeholder Get API keys from your KhudeBarta Portal. Contact their support for Sender ID (masking) approval.

Credentials You Need

Credential Where to Get Placeholder
API Key KhudeBarta Portal → API Settings YOUR_KB_API_KEY
Secret Key KhudeBarta Portal → API Settings YOUR_KB_SECRET_KEY
Sender ID (Caller ID) Apply for masking via KhudeBarta YOUR_SENDER_ID

API Reference

GET http://portal.khudebarta.com:3775/sendtext

Param Type Required Description
apikey String REQUIRED Your API Key
secretkey String REQUIRED Your Secret Key
callerID String REQUIRED Sender ID / masking name
toUser String REQUIRED Phone with 88 prefix (e.g. 8801712345678)
messageContent String REQUIRED SMS message body

cURL Example

curl -G "http://portal.khudebarta.com:3775/sendtext" \
  --data-urlencode "apikey=YOUR_KB_API_KEY" \
  --data-urlencode "secretkey=YOUR_KB_SECRET_KEY" \
  --data-urlencode "callerID=YOUR_SENDER_ID" \
  --data-urlencode "toUser=8801712345678" \
  --data-urlencode "messageContent=Your OTP is 123456"

Phone Number Formatting (Bangladesh)

// Always ensure phone starts with "88"
function formatBDPhone(phone) {
  phone = phone.replace(/\D/g, '');          // Remove non-digits
  if (!phone.startsWith('88')) phone = '88' + phone;
  return phone;  // "01712345678" → "8801712345678"
}

🗄️ Database Schema

payment_gateways (for SSLCOMMERZ & bKash)

CREATE TABLE payment_gateways (
  id             BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  provider_name  VARCHAR(255) NOT NULL,  -- 'ssl_commerz' | 'bkash'
  api_key        VARCHAR(255),           -- SSLC: Store ID | bKash: App Key
  secret_key     VARCHAR(255),           -- SSLC: Store Pass | bKash: App Secret
  username       VARCHAR(255),           -- bKash only: username
  password       VARCHAR(255),           -- bKash only: password
  live           TINYINT DEFAULT 0,      -- 0=Sandbox, 1=Live
  status         TINYINT DEFAULT 1,      -- 0=Inactive, 1=Active
  created_at     TIMESTAMP NULL,
  updated_at     TIMESTAMP NULL
);

sms_gateways (for KhudeBarta)

CREATE TABLE sms_gateways (
  id             BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  provider_name  VARCHAR(255) NOT NULL,  -- 'KhudeBarta'
  api_endpoint   VARCHAR(255) NOT NULL,  -- 'http://portal.khudebarta.com:3775/sendtext'
  api_key        VARCHAR(255) NOT NULL,
  secret_key     VARCHAR(255),
  sender_id      VARCHAR(255) NOT NULL,  -- Caller ID / masking
  status         TINYINT DEFAULT 0,      -- 0=Inactive, 1=Active
  created_at     TIMESTAMP NULL,
  updated_at     TIMESTAMP NULL
);

🚀 Integration Checklist for OTT Hub

✅ This guide is language-agnostic All API calls are shown as raw HTTP/cURL — implement in any backend: Node.js, Python, PHP, .NET, Go, etc.

Step 1: SSLCOMMERZ

  1. Register at developer.sslcommerz.com (sandbox) or contact for live
  2. Get YOUR_SSLC_STORE_ID & YOUR_SSLC_STORE_PASSWORD
  3. Create payment_gateways table (schema above)
  4. Implement POST to /gwprocess/v4/api.php
  5. Handle success/fail/cancel callback URLs (exclude from CSRF)
  6. Validate payment server-side via val_id

Step 2: bKash

  1. Register at developer.bka.sh for sandbox
  2. Get YOUR_BKASH_APP_KEY, YOUR_BKASH_APP_SECRET, YOUR_BKASH_USERNAME, YOUR_BKASH_PASSWORD
  3. Implement token grant → cache for 55 min
  4. Implement create → redirect to bkashURL → callback → execute
  5. Verify: statusCode == "0000" && transactionStatus == "Completed"

Step 3: KhudeBarta SMS

  1. Contact KhudeBarta for API credentials
  2. Get YOUR_KB_API_KEY, YOUR_KB_SECRET_KEY, YOUR_SENDER_ID
  3. Create sms_gateways table (schema above)
  4. Send SMS via HTTP GET to portal.khudebarta.com:3775/sendtext
  5. Ensure phone numbers start with 88
